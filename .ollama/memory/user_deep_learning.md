**Preference**: Learn every concept to the bone — deep, foundational understanding from first principles so they can build independently without asking for help.

**Why**: User wants the mental models, mechanisms, and "how things actually work" so they can extend/adapt code without guidance.

**How to apply in every session**:
- Explain concepts from first principles, not just surface usage patterns
- Reveal what's happening under the hood in concrete, mechanistic terms
- Include the "why" behind design decisions, API choices, and abstractions
- Connect new topics to things they already know (build on existing mental models)
- Provide enough depth that the user can infer related patterns and build extensions without step-by-step instructions
- Avoid hand-waving — break abstractions down into what they actually do at the machine/OS/language level
- When showing an example, trace through it line-by-line with actual values
- Treat every session as a continuation of this deep-learning journey — refer to past phases/concepts and show how they connect
